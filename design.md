# TaskLite

## 1.2 目的

Windows標準の Task Manager よりも軽量・高速で、Linux系UI思想を取り入れたシステムモニタを提供する。

## 1.3 背景課題

* 標準Task Managerは機能過多
* メモリ使用量が比較的高い
* UIがWindows固有思想
* 開発学習目的として内部API理解を深めたい

---

# 2. スコープ定義

## 2.1 対象OS

* Windows 11 (優先)

---

## 2.2 除外機能

* スタートアップ管理
* サービス管理
* パフォーマンス履歴の長期保存
* 自動アップデート機構
* UWP統合

---

# 3. 機能要件

## 3.1 プロセス表示

### F-01

現在実行中のプロセス一覧を表示する

### F-02

以下の項目を表示：

* PID
* プロセス名
* CPU使用率 (%)
* メモリ使用量 (MB)
* 実行ユーザー（将来拡張）

### F-03

CPU使用率降順で並び替え

### F-04

Top N（初期値20）表示

---

## 3.2 リソース表示タブ

Linux系参考：

* GNOME System Monitor
* KDE System Monitor

### F-05

全体CPU使用率表示

### F-06

メモリ使用率表示

### F-07

軽量リアルタイム線グラフ（最大30秒履歴）

---

## 3.3 更新制御

### F-08

更新間隔 1000ms以上

### F-09

更新間隔変更可能（1000ms〜5000ms）

---

## 3.4 操作機能

### F-10

プロセス終了（管理者権限必要）

### F-11

検索フィルタ

---

# 4. 非機能要件（最重要）

## 4.1 パフォーマンス

| 項目     | 目標      |
| ------ | ------- |
| 常駐メモリ  | 25MB以下  |
| CPU使用率 | 平均1%未満  |
| 起動時間   | 1秒以内    |
| 更新処理時間 | 100ms未満 |

---

## 4.2 軽量設計原則

* 不要なnpm依存禁止
* アニメーション最小化
* グラフライブラリは最軽量選択
* 差分更新のみ

---

## 4.3 セキュリティ

* 任意コード実行禁止
* 管理者権限昇格は明示的
* 外部通信なし（完全ローカル）

---

# 5. アーキテクチャ要件

## 5.1 技術スタック

* Tauri v2
* Rust backend
* Svelte frontend
* sysinfoクレート（システム情報取得）

---

## 5.2 レイヤー構成

```
UI Layer (WebView)
  ↓ invoke
Rust Command Layer
  ↓
System Data Layer
  ↓
Windows API / sysinfo
```

---

# 6. 品質要件
